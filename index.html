Key Candy

Makes your web app melt in your hands, not on your mouse.

Download

  • keyCandy.js - Uncompressed

Like magic? Prepare to be amazed!

The Quick Demo:

Step 1. put "accesskey" attributes on your elements.

<p><a href="#" accesskey="A">An Anchor</a></p>
<form>
    <p><button accesskey="B">A Button</button></p>    
    <p>
        <label accesskey="C" for="text_input">A text input</label>
        <input id="text_input" name="text_input" type="text" />
    </p>
    <p>
        <label accesskey="H" for="check_it">A Checkbox: <input id="check_it" type="checkbox" /></label>
    </p>
    <p>
        <label for="comment" accesskey="T">A Textarea</label>
        <textarea id="comment"></textarea>
    </p>
    <p>
        <label for="choices" accesskey="S">Choose one: </label>
        <select id="choices">
            <option>First</option>
            <option>Second</option>
        </select>
    </p>
</form>

Step 2. - Create Your Tooltip Styles

/* 
Example CSS styles for Key Candy tooltips 
These are included in keyCandy.css in the source archive.
Place these rules in your main stylesheet or a <style> tag in the head
of your HTML document.
*/
.keycandy:before,.keycandy *[accesskey]:before {
	background:#ffa;
	position:absolute;
	font:16px/24px Arial,Helvetica,sans-serif;
	vertical-align:middle;
	color:#3f3f3f;
	text-align:center;
	border:1px solid #ccc;
	-moz-border-radius:8px;
	-webkit-border-radius:8px;
	border-radius:8px;
	-moz-box-shadow:0 0 3px #bababa;
	-webkit-box-shadow:0 0 3px #bababa;
	box-shadow:0 0 3px #bababa;
	text-decoration:none;
	z-index:999
}

.keycandy:before {
	top:2px;
	content:"Press [esc] to clear tooltips."
}

.keycandy *[accesskey] {
	position:relative
}

.keycandy *[accesskey]:before {
	right:-8px;
	bottom:-8px;
	content:attr(accesskey);
	width:24px;
	height:24px
}

Step 3. - init Key Candy JavaScript


KeyCandy.init();

What else do I need to know?

Not much really.

  • Replaced elements (e.g., img, input, select, textarea) do not have the :before and :after pseudo-elements. so you have to "workaround" that by...using best practices! If you are using proper labels for your form elements, you can simply add the accesskey attribute to the label.
  • Currently, Key Candy depends on jQuery (available via Google CDN.) for class manipulations and event handling
  • So, just include jQuery and keyCandy.js (or keyCandy.min.js).

Why would we need this?

Because people are using your web app to do something and it should help them do that something as much as it can. Traditional desktop applications help users by giving them keyboard shortcuts for common actions. Now you can do the same in your app (with a little added sugar). The extra sweetness is the tooltips that Key Candy displays for your users when they press [Ctrl] on Windows, Linux or Mac or Command on Mac.
